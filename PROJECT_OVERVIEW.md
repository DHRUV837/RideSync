# Carpooling App Project Structure

## Overview
This is a comprehensive carpooling platform with AI-powered ride matching and route optimization.

## Key Features Implemented
✅ **Authentication**: JWT-based login/registration
✅ **User Management**: Rider, Driver, Admin roles
✅ **Ride Management**: Create rides, view available rides
✅ **Database**: PostgreSQL with comprehensive schema
✅ **ML Service**: K-Means clustering, route optimization
✅ **Docker**: Complete containerized setup
✅ **API**: RESTful endpoints with error handling
✅ **Frontend Integration**: API service layer with axios

## Project Statistics
- **Backend Classes**: 15+ (Entities, Repositories, Services, Controllers)
- **Frontend Components**: 10+ (Pages, Layouts, Components)
- **Database Tables**: 8 (Users, Profiles, Rides, Bookings, Ratings, Complaints, Stops)
- **API Endpoints**: 20+ (Auth, Rides, ML endpoints)
- **ML Algorithms**: K-Means, Nearest Neighbor for optimization
